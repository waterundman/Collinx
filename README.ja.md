<h1 align="center">Collinx</h1>

<p align="center">

---

Collinxはボタンを押してくれません。あなたの音楽プロジェクトを型付きプロパティグラフに変換し、8つの専門エージェントがそのグラフを直接操作します——作曲、編曲、オーケストラ、ミキシング、記譜、教育、審美学習。すべての操作は構造化されたDiffEnvelopeを生成します。承認すればメインブランチに書き込まれます。不要ならロールバック。

これはDAWをラップしたチャットボットではありません。エージェントにはツール権限、操作監査、ドメイン説明能力があります。やることすべて——検証可能、追跡可能、改善を教えられます。

## 仕組み

```
┌─────────────────────────────────────────────────────────┐
│                     Project Graph                       │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────┐ │
│  │Composition│  │ Notation │  │Performance│  │  Audio  │ │
│  │  Layer    │→│  Layer   │→│   Layer   │→│  Layer  │ │
│  │何が書かれた│  │どう見える│  │どう演奏   │  │どう聞こ │ │
│  │か         │  │か        │  │されるか   │  │えるか   │ │
│  └──────────┘  └──────────┘  └──────────┘  └─────────┘ │
│       ↑              ↑              ↑            ↑      │
│  ┌────┴──────────────┴──────────────┴────────────┴────┐ │
│  │              Agent Bus (pub/sub)                   │ │
│  └────┬──────┬──────┬──────┬──────┬──────┬──────┬────┘ │
│     Plan  Compose Arrange Orch.  Mix  Engr. Teach  Taste│
└─────────────────────────────────────────────────────────┘
         │                                    │
         ▼                                    ▼
   DiffEnvelope                        Taste Genome
   (提案 → 承認 → 書込)            (ベイズ審美記憶)
```

**Project Graph** — 12ノードタイプ、10エッジタイプ、バージョン管理とブランチ管理対応。すべての音楽データがここに存在します。

**DiffEnvelope** — すべてのエージェント操作の出力フォーマット。サマリー、ドメイン説明、証拠参照、リスクフラグ、ロールバックトークンを含みます。デフォルト権限：`proposal_only`——エージェントは提案のみ、あなたが決めます。

**Taste Genome** — 3層にわたるあなたの審美記憶：説明可能層（人間が読めるパラメータ）、分布層（Beta/Dirichlet/Bernoulli）、埋め込み層（タグベクトル）。和声、旋律、リズム、テクスチャー、ティンバー、フォーム、ミックス、リジェクトの8ドメインをカバー。使うほどにエージェントが理解します。

## 8つのエージェント

| | エージェント | やりこと |
|---|---|---|
| 1 | **Planner** | 「サビを書いて」をタスクグラフに分解、具体的ツールにマッピング |
| 2 | **Composer** | メロディとハーモニーを生成 — 13スケール、6スタイル、8リズムテンプレート |
| 3 | **Arranger** | モチーフ → 完全なアレンジ。フォーム構造、エネルギーカーブ、バリエーション生成 |
| 4 | **Orchestrator** | 楽器編成 — 30+楽器、7ファミリー、レジスタ競合検出、演奏可能性検証 |
| 5 | **Engraving** | スコアレイアウト、衝突検出、パート抽出（4ハウススタイル） |
| 6 | **Mixing** | ゲイン構造、FXチェーン、パンニング — ルールベース、ブラックボックスではない |
| 7 | **Teaching** | すべての音楽的意思決定を4難易度で説明 |
| 8 | **Taste Memory** | エクスポートを分析、審美的証拠を蓄積、tasteで候補をランク付け |

すべてのエージェント操作はDiffEnvelope——検証、承認、拒否、ロールバックが可能。

## UI — 9タブ

| Tab | やりこと |
|---|---|
| **compose** | ピアノロール + コンパクトスコア + アレンジメントタイムライン |
| **arrange** | セクションアレンジ + Diffプレビュー |
| **orchestrate** | 楽器割り当て + 競合表示 |
| **mixer** | フルミキサー — ゲイン、パン、ミュート、ソロ、FXチェーン |
| **score** | フルスコア — 自動レイアウト、パート抽出、MusicXMLエクスポート |
| **taste** | Tasteゲノムタイムライン + パラメータライブラリ + Diffレポート |
| **teaching** | マルチレベル音楽理論説明 |
| **agent** | Diffキュー + チャットインターフェース |
| **graph** | プロジェクトグラフ可視化 — Canvas 2D力指向レイアウト |

## クイックスタート

```bash
# 必要要件：Node.js >= 18, pnpm 9

pnpm install          # 依存関係のインストール
pnpm dev              # 全開発サーバー起動
pnpm dev:ui           # UIのみ起動（ポート5180）
pnpm build            # 全パッケージビルド
pnpm test             # ユニットテスト（Vitest）
pnpm test:e2e         # E2Eテスト（Playwright、84ケース）
pnpm typecheck        # 型チェック
```
